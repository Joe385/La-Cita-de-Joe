Use Attibute Selectors to Style Elements

Other ways to style custom group elements is through Attribute selectors

1
2
3		 
[type='radio'] {
    margin: 20px 0px 20px 0px;
}

An attribute selector ([attr=value]) matches and styles elements that have a specific attribute value.
For example in the code above, it will find elements that have the attribute [type='radio'] and give it the custom margins.

Understand Absolute versus Relative Units

CSS has a lot of other measurements to use however, the two main types are absolute and relative.
Absolute units like in, mm, and px approximate the actual measurement on-screen.
Relative unlits like em, rem, and % are relative to another length's value. For example, em is based on the size of an element's font. So if you set a font-size property of a child, it will be relative to its parent's font-size
Although briefly explained, it will be further covered in Responsive Web Design Principles section.

Override Styles in Subsequent CSS

The second declaration takes precedence over the first.

1 
<h1 class="losingclass winningclass">Test</h1>

Simply, winningclass's style will be used instead of losingclass.

Override Class Declarations by Styling ID Attributes

ID's override classes

1 
<h1 class="losingclass" id="winningid">Test</h1>

winningid's style will be used instead of losingclass.

Override Class Declarations with Inline Styles

Inline Styles (Style Attributes) overrides class declarations & ids.

1 
<h1 style="color: white;" class="losingclass" id="losingid">Test</h1>

The inline style (white) will be used instead of losingclass & losingid.

Override All Other Styles by using Important

!important overrides all other styles

1
2
3
4		 
/*Inside .cs file*/
.notlosingclass {
    color: #000000 !important;
}
1
2
3		 
<!-- Inside .html file-->
<h1 style="color: white;" class="notlosingclass" id="losingid">Test</h1>
<!--Make sure to reference the .cs file for this to work-->

Due to !important, notlosingclass's style will be used despite it having the lower specificity (class).
"In many situations, you will use CSS libraries. These may accidentally override your own CSS. So when you absolutely need to be sure that an element specific CSS, you can use !important." - freecodecamp

Use Hex Code for Specific Colors

You can represent colors using hexadecimal code a.k.a. hex code.
As you may know, hexadecimal code use base 16 (0-9, A-F)
For example, black can also be represented as #000000

Use Hex Code to Mix Colors

Hex codes use 6 hexadecimal digits 2 red, 2 green, and 2 blue.
(#1E90FF)
Mixing these colors could create over 16 million colors.
0 is the complete absence of color and F is the maximum possible brightness

Use Abbreviated Hex Code

You can shorten hex codes to 3 digits: #F1C, reducing the number of colors to 4,000.

Use RGB values to Color Elements

You can use rgb values to represent colors

1 
background-color: rgb(0, 0, 0)

It's similar to 3 letter hex codes.

Use CSS Variables to change several elements at once

Intro to using CSS variables. However, example just changes colors of multiple elements.

Create a custom CSS Variable

Sample CSS variable
1 
--example-variable: gray;

Use a custom CSS Variable

Sample variable usage

1 
background-color: var(--example-variable);

Referencing the variable above will give elements the value of that variable, which is gray.

Attach a Fallback value to a CSS Variable

Similar to degrading text, you should have a fallback variable if a browser can't reference a variable.
"This fallback is not used to increase browser compatibility, and it will not work on IE browsers. Rather, it is used so that the browser has a color to display if it cannot find your variable." -freecodecamp

1 
background-color: var(--example-variable, fallbackcolor);

Improve Compatibility with Browser Fallbacks

In order to compensate for browser compatibility issues, you should implement fallbacks.
For example, "Internet Explorer will ignore the background color because it does not support CSS variables...it will revert to the default value, which is typically not ideal" -freecodecamp.
So an easy way to compensate for this is to implement other values before variables.

1
2
3
4
5		 
.red-box {
    --example-variable: red;
    background: red;/*This is what you should do*/
    background: var(--example-variable);
}

Inherit CSS Variables

Creating a variable only allows it to be used within a selector (container) and its descendents. To create global variables, make use of the :root element.

1
2
3
4
5
6
7
8		 
root: {
    --example-variable: red;/*This is root*/
}

.red-box {
    background: red;
    background: var(--example-variable);
}

":root is a pseudo-class selector that matches the root element of the document, usually the html element...they will be available globally and can be accessed from any other selector"-freecodecamp

Change a variable for a specific area

You can change a color of a selector by reassigning variables within a class. As such you can create flexible variables.

1
2
3
4
5
6
7
8		 
root: {
    --example-variable: red;/*example-variable is red*/
}

.red-box {
    background: var(--example-variable);/*red box*/
    --example-variable: blue;/*example-variable is blue*/
}
.blue-box {
    background: var(--example-variable);/*blue box*/
}
